## 4. Installation & Setup Instructions

### Step 1: Create a Supabase Account
1.  Go to [supabase.com](https://supabase.com/) and sign up.
2.  Click **"New Project"**.
3.  Enter a Name (e.g., `IdeaBoard`) and a Database Password.
4.  Select a Region close to you.
5.  Click **"Create new project"**.

### Step 2: Create the Database Table
1.  In your project dashboard, go to the **SQL Editor** (icon looks like `>_` on the left).
2.  Paste and run the following SQL code to create the table and set up security:

```sql
-- Create the table
CREATE TABLE ideas (
  id bigint generated by default as identity primary key,
  content text,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- Turn on security
ALTER TABLE ideas ENABLE ROW LEVEL SECURITY;

-- Create a policy that lets ANYONE read and write (Toolkit Mode)
CREATE POLICY "Enable public access" ON ideas FOR ALL
USING (true)
WITH CHECK (true); -- Add the WITH CHECK clause here
```

### Step 3: Get API Keys
1.  Go to **Project Settings** (Gear icon) > **API**.
2.  Copy the `Project URL` and the `anon` `public` Key.

### Step 4: Configure the Application
1.  Open `index.html`.
2.  Replace `SUPABASE_URL` and `SUPABASE_KEY` with your copied values.

## 5. Minimal Working Example
The `index.html` file is a self-contained application. It connects to Supabase to manage a list of ideas.

**Key Code Snippet (Data Fetching):**
```javascript
// Initialize Supabase

// Fetch data from 'ideas' table
async function fetchIdeas() {
    const { data, error } = await supabase
        .from('ideas')
        .select('*')
        .order('created_at', { ascending: false });
    
    // ... render data to UI ...
}
```

**Expected Output:**

*   An input field to type ideas.
*   A list of saved ideas below it.
*   Clicking "Save" adds the idea to the list and database instantly.
*   Clicking the red "X" removes the idea.

## 7. Common Issues & Fixes
**Issue 1: "new row violates row-level security policy"**
*   **Cause**: The RLS policy didn't allow writing (INSERT/DELETE) for anonymous users.
*   **Fix**: Updated the SQL policy to include `WITH CHECK (true)` which explicitly allows new rows to be inserted.

**Issue 2: Connection Errors**
*   **Cause**: Incorrect API URL or Key.
*   **Fix**: Double-checked the values in `Project Settings > API` and ensured no extra spaces were pasted.

**Issue 3: Delete button not working**
*   **Cause**: Missing RLS policy for DELETE or incorrect ID passing.
*   **Fix**: The policy `FOR ALL` covers DELETE, and we ensured the `deleteIdea(id)` function receives the correct row ID.
